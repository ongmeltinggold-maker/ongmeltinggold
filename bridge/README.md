# ONG Card Reader Bridge (เครื่องอ่านบัตรประชาชน)

โปรแกรมเล็ก ๆ รันบน **เครื่องหน้าร้าน** ที่ต่อเครื่องอ่านบัตร Smart Card (PC/SC)
อ่านชิปบัตรประชาชนไทยแล้วเปิดพอร์ต `http://127.0.0.1:8765/read` ให้หน้าเว็บดึงข้อมูลมาเติมฟอร์มอัตโนมัติ
(แทนวิธี keyboard-emulation ของ Siam ID เดิม — ปลอดภัยกว่า ส่งเป็น JSON และทำ audit ได้)

## ติดตั้ง
```bash
pip install pyscard
# Linux เพิ่ม: sudo apt install pcscd libpcsclite-dev && sudo systemctl start pcscd
# Windows/macOS: มี PC/SC ในตัว
```

## ใช้งาน
```bash
python card_reader.py     # เปิดค้างไว้ระหว่างเปิดร้าน
```
จากนั้นในหน้า "เพิ่มลูกค้า (KYC)" กดปุ่ม **"🪪 อ่านบัตร"** → ระบบดึงข้อมูลจากชิปมาเติมฟอร์ม
(เลขบัตร, ชื่อไทย/อังกฤษ, วันเกิด, ที่อยู่, วันออก/หมดอายุบัตร)

## ทดสอบโดยไม่ต้องมีเครื่องอ่าน (Mock)
```bash
python card_reader.py --mock      # คืนข้อมูลตัวอย่าง (ไม่แตะฮาร์ดแวร์)
```
เปิดหน้า "เพิ่มลูกค้า" กด "🪪 อ่านบัตร" → ฟอร์มจะถูกเติมด้วยข้อมูลตัวอย่าง (ใช้ตรวจว่าเว็บ↔bridge เชื่อมกันถูก)

## ขั้นตอนทดสอบกับเครื่องจริง (Checklist)
1. เสียบเครื่องอ่าน USB + เสียบบัตรประชาชน
2. Linux: `sudo systemctl status pcscd` ต้อง active · Windows/macOS ข้ามได้
3. `pip install pyscard` แล้ว `python card_reader.py` (โหมดจริง)
4. เปิด `http://127.0.0.1:8765/read` ในเบราว์เซอร์ → ควรเห็น JSON ข้อมูลบัตร
5. ในเว็บกด "🪪 อ่านบัตร" → ฟอร์มเติมอัตโนมัติ (เลขบัตร/ชื่อ/ที่อยู่/วันเกิด)
6. ถ้า error: ตรวจไดรเวอร์เครื่องอ่าน, สิทธิ์ USB, และ pcscd (Linux)

## หมายเหตุ
- รองรับเครื่องอ่านมาตรฐาน PC/SC (Zoweetek, HawkEye ฯลฯ)
- ถ้าไม่พบเครื่องอ่าน จะคืน `{"ok":false,"error":"..."}` และหน้าเว็บจะแจ้งเตือน (ยังกรอกมือได้ปกติ)
- อ่านข้อความด้วย TIS-620 → แปลงเป็น UTF-8; รูปถ่ายจากชิป (optional) ยังไม่เปิดใช้เพื่อความเร็ว
- APDU offset ของบัตรบางรุ่นอาจต่างเล็กน้อย — ปรับใน `CMD` ได้
